    Install EDR to Server/Endpoint

    Panduan lengkap untuk menginstal agen SentinelOne EDR pada server dan endpoint dengan berbagai sistem operasi.

    Overview

    SentinelOne EDR agent adalah komponen utama yang perlu diinstal pada setiap endpoint atau server yang akan dimonitor. Agent ini bertanggung jawab untuk mengumpulkan telemetri, mendeteksi ancaman, dan menjalankan respons otomatis.

    Prerequisites

    Sebelum melakukan instalasi, pastikan persyaratan berikut terpenuhi:

    System Requirements

    • Windows 10/11, Windows Server 2016/2019/2022
    • RAM minimum 2GB, recommended 4GB+
    • Disk space minimum 2GB
    • Administrator privileges
    • Ubuntu 18.04+, CentOS 7+, RHEL 7+, SLES 12+
    • RAM minimum 1GB, recommended 2GB+
    • Disk space minimum 1GB
    • Root privileges
    • macOS 10.14+
    • RAM minimum 2GB, recommended 4GB+
    • Disk space minimum 2GB
    • Administrator privileges

    Network Requirements

    • Outbound HTTPS (443) ke SentinelOne Management Console
    • DNS resolution untuk domain SentinelOne
    • Firewall configuration untuk komunikasi agent

    Persiapan Instalasi

    1. Unduh Paket Instalasi

    • Masuk ke SentinelOne Management Console
    • Pergi ke Sentinels 3e Install New Sentinel
    • Pilih platform (Windows/Linux/macOS)
    • Unduh paket instalasi yang sesuai

    2. Persiapkan Installation Token

    • Salin Site Token dari Console
    • Ini digunakan untuk menghubungkan agent ke management console
    • Simpan token dengan aman untuk proses instalasi

    Instalasi Berdasarkan Platform

    Metode 1: Command Line Installation

    Instalasi Silent dengan MSI: 

    msiexec /i SentinelInstaller_windows_64bit.msi /quiet SITE_TOKEN=your_site_token_here

    Instalasi dengan Parameter Tambahan: 

    msiexec /i SentinelInstaller_windows_64bit.msi /quiet SITE_TOKEN=your_site_token_here REBOOT=ReallySuppress

    Metode 2: GUI Installation

    1. Jalankan file installer SentinelInstaller_windows_64bit.msi
    2. Ikuti wizard instalasi
    3. Masukkan Site Token ketika diminta
    4. Tunggu hingga instalasi selesai
    5. Restart sistem jika diperlukan

    Metode 3: PowerShell Installation

    # Download installer
$installer = "SentinelInstaller_windows_64bit.msi"
$token = "your_site_token_here"

# Install silently
Start-Process msiexec.exe -ArgumentList "/i $installer /quiet SITE_TOKEN=$token" -Wait

    Verifikasi Instalasi

    "C:\Program Files\SentinelOne\Sentinel Agent\SentinelCtl.exe" status

    Metode 1: RPM-based Systems (CentOS/RHEL/Fedora)

    Instalasi langsung: 

    # Install package
sudo rpm -i SentinelAgent_linux.rpm

# Set site token
sudo /opt/sentinelone/bin/sentinelctl management token set YOUR_SITE_TOKEN

# Start agent
sudo /opt/sentinelone/bin/sentinelctl control start

    Instalasi dengan yum/dnf: 

    # Install using yum
sudo yum localinstall SentinelAgent_linux.rpm

# Or using dnf (newer systems)
sudo dnf install SentinelAgent_linux.rpm

# Configure and start
sudo /opt/sentinelone/bin/sentinelctl management token set YOUR_SITE_TOKEN
sudo /opt/sentinelone/bin/sentinelctl control start

    Metode 2: DEB-based Systems (Ubuntu/Debian)

    Instalasi langsung: 

    # Install package
sudo dpkg -i SentinelAgent_linux.deb

# Fix dependencies if needed
sudo apt-get install -f

# Set site token
sudo /opt/sentinelone/bin/sentinelctl management token set YOUR_SITE_TOKEN

# Start agent
sudo /opt/sentinelone/bin/sentinelctl control start

    Instalasi dengan apt: 

    # Install using apt
sudo apt install ./SentinelAgent_linux.deb

# Configure and start
sudo /opt/sentinelone/bin/sentinelctl management token set YOUR_SITE_TOKEN
sudo /opt/sentinelone/bin/sentinelctl control start

    Metode 3: Generic Installation Script

    #!/bin/bash
# SentinelOne Installation Script

SITE_TOKEN="YOUR_SITE_TOKEN"
INSTALLER_PATH="/path/to/SentinelAgent_linux.rpm"  # or .deb

# Detect package manager
if command -v rpm 3e /dev/null 23e&1; then
    sudo rpm -i "$INSTALLER_PATH"
elif command -v dpkg 3e /dev/null 23e&1; then
    sudo dpkg -i "$INSTALLER_PATH"
    sudo apt-get install -f  # Fix dependencies
fi

# Configure agent
sudo /opt/sentinelone/bin/sentinelctl management token set "$SITE_TOKEN"
sudo /opt/sentinelone/bin/sentinelctl control start

# Verify installation
sudo /opt/sentinelone/bin/sentinelctl status

    Verifikasi Instalasi

    sudo /opt/sentinelone/bin/sentinelctl status
sudo systemctl status sentinelone

    Metode 1: Command Line Installation

    Instalasi dengan installer command: 

    # Install package
sudo installer -pkg SentinelAgent_macos.pkg -target /

# Set site token
sudo /Library/Sentinelone/sentinel/bin/sentinelctl management token set YOUR_SITE_TOKEN

# Start agent
sudo /Library/Sentinelone/sentinel/bin/sentinelctl control start

    Metode 2: GUI Installation

    1. Double-click file SentinelAgent_macos.pkg
    2. Ikuti wizard instalasi macOS
    3. Masukkan password administrator ketika diminta
    4. Buka Terminal setelah instalasi
    5. Set site token: 
      sudo /Library/Sentinelone/sentinel/bin/sentinelctl management token set YOUR_SITE_TOKEN
sudo /Library/Sentinelone/sentinel/bin/sentinelctl control start

    Metode 3: Automated Script

    #!/bin/bash
# SentinelOne macOS Installation Script

SITE_TOKEN="YOUR_SITE_TOKEN"
PKG_PATH="/path/to/SentinelAgent_macos.pkg"

# Install package
sudo installer -pkg "$PKG_PATH" -target /

# Wait for installation to complete
sleep 10

# Configure agent
sudo /Library/Sentinelone/sentinel/bin/sentinelctl management token set "$SITE_TOKEN"
sudo /Library/Sentinelone/sentinel/bin/sentinelctl control start

# Verify installation
sudo /Library/Sentinelone/sentinel/bin/sentinelctl status

    Verifikasi Instalasi

    sudo /Library/Sentinelone/sentinel/bin/sentinelctl status
sudo launchctl list | grep sentinelone

    4. Verifikasi Instalasi

    • Windows: 

      "C:\Program Files\SentinelOne\Sentinel Agent\SentinelCtl.exe" status

    • Linux: 

      sudo /opt/sentinelone/bin/sentinelctl status

    • macOS: 

      sudo /Library/Sentinelone/sentinel/bin/sentinelctl status

    • Console:

    • Cek di Sentinels untuk keterhubungan endpoint
    • Status harus Online dan Protected

    Troubleshooting Umum

    • Agent Tidak Muncul di Console
    • Penyebab: Token salah atau masalah jaringan

    • Solusi: Pastikan token benar dan periksa koneksi

    • Gagal Instalasi

    • Penyebab: Privilege tidak cukup

    • Solusi: Jalankan sebagai Administrator/root

    • Agent Offline

    • Penyebab: Firewall memblokir koneksi

    • Solusi: Periksa pengaturan firewall dan jaringan

    • Penggunaan CPU Tinggi

    • Penyebab: Pemindaian real-time
    • Solusi: Atur exclusion untuk aplikasi kritis

    Uninstall SentinelOne EDR Agent

    Penting

    Proses uninstall akan menghapus semua proteksi pada endpoint. Pastikan Anda memiliki alternatif keamanan sebelum melakukan uninstall.

    Metode 1: Command Line Uninstall

    Menggunakan SentinelCtl: 

    "C:\Program Files\SentinelOne\Sentinel Agent\SentinelCtl.exe" uninstall

    Uninstall dengan passphrase (jika ada): 

    "C:\Program Files\SentinelOne\Sentinel Agent\SentinelCtl.exe" uninstall --passphrase "your_passphrase"

    Metode 2: MSI Uninstall

    Find product code and uninstall: 

    # Find SentinelOne product code
wmic product where "name like '%SentinelOne%'" get identifyingnumber,name

# Uninstall using product code
msiexec /x {PRODUCT-CODE} /quiet

    Metode 3: Programs and Features

    1. Buka Control Panel 3e Programs and Features
    2. Cari "SentinelOne" dalam daftar
    3. Klik Uninstall
    4. Ikuti wizard uninstall

    Metode 4: PowerShell Uninstall

    # Get SentinelOne installation
$sentinelone = Get-WmiObject -Class Win32_Product | Where-Object {$_.Name -like "*SentinelOne*"}

# Uninstall
if ($sentinelone) {
    $sentinelone.Uninstall()
    Write-Host "SentinelOne uninstalled successfully"
}

    Metode 1: SentinelCtl Uninstall

    Standard uninstall: 

    sudo /opt/sentinelone/bin/sentinelctl uninstall

    Uninstall dengan passphrase: 

    sudo /opt/sentinelone/bin/sentinelctl uninstall --passphrase "your_passphrase"

    Metode 2: Package Manager Uninstall

    RPM-based systems: 

    # Find package name
rpm -qa | grep -i sentinel

# Remove package
sudo rpm -e SentinelAgent

# Or using yum/dnf
sudo yum remove SentinelAgent
sudo dnf remove SentinelAgent

    DEB-based systems: 

    # Find package name
dpkg -l | grep -i sentinel

# Remove package
sudo dpkg -r sentinelagent

# Or using apt
sudo apt remove sentinelagent
sudo apt purge sentinelagent  # Remove config files too

    Metode 3: Force Removal Script

    #!/bin/bash
# Force removal of SentinelOne (use with caution)

echo "Stopping SentinelOne services..."
sudo systemctl stop sentinelone
sudo systemctl disable sentinelone

echo "Removing SentinelOne files..."
sudo rm -rf /opt/sentinelone/
sudo rm -rf /etc/sentinelone/
sudo rm -f /etc/systemd/system/sentinelone.service

echo "Cleaning up systemd..."
sudo systemctl daemon-reload

echo "SentinelOne force removal completed"

    Metode 1: SentinelCtl Uninstall

    Standard uninstall: 

    sudo /Library/Sentinelone/sentinel/bin/sentinelctl uninstall

    Uninstall dengan passphrase: 

    sudo /Library/Sentinelone/sentinel/bin/sentinelctl uninstall --passphrase "your_passphrase"

    Metode 2: Manual Removal

    # Stop SentinelOne service
sudo launchctl unload /Library/LaunchDaemons/com.sentinelone.*

# Remove SentinelOne files
sudo rm -rf /Library/Sentinelone/
sudo rm -f /Library/LaunchDaemons/com.sentinelone.*
sudo rm -rf /Library/Application\ Support/SentinelOne/

    Metode 3: Uninstall Script

    #!/bin/bash
# SentinelOne macOS Uninstall Script

echo "Uninstalling SentinelOne from macOS..."

# Try sentinelctl first
if [ -f "/Library/Sentinelone/sentinel/bin/sentinelctl" ]; then
    echo "Using sentinelctl for removal..."
    sudo /Library/Sentinelone/sentinel/bin/sentinelctl uninstall
else
    echo "Manual removal..."
    sudo launchctl unload /Library/LaunchDaemons/com.sentinelone.* 23e/dev/null
    sudo rm -rf /Library/Sentinelone/
    sudo rm -f /Library/LaunchDaemons/com.sentinelone.*
    sudo rm -rf /Library/Application\ Support/SentinelOne/
fi

echo "SentinelOne uninstall completed"

    Verifikasi Uninstall

    # Check if services are removed
sc query SentinelAgent
sc query SentinelHelperService

# Check if files are removed
dir "C:\Program Files\SentinelOne"

    # Check if package is removed
rpm -qa | grep -i sentinel
dpkg -l | grep -i sentinel

# Check if services are stopped
sudo systemctl status sentinelone

# Check if files are removed
ls -la /opt/sentinelone/

    # Check if services are removed
sudo launchctl list | grep sentinelone

# Check if files are removed
ls -la /Library/Sentinelone/

    Troubleshooting Uninstall

    Common Issues

    • Password/Passphrase Required: Hubungi administrator untuk mendapatkan passphrase uninstall
    • Service Won't Stop: Force stop service sebelum uninstall
    • Files Still Present: Manual removal mungkin diperlukan
    • Permission Denied: Pastikan running sebagai admin/root
    • Connection Lost During Uninstall: Lihat bagian Decommissioning di bawah

    Decommissioning Agent from Console

    Uninstall dengan Gangguan Koneksi

    Ketika terjadi gangguan koneksi internet saat proses uninstall, agent mungkin sudah terhapus dari endpoint tetapi masih tercatat online atau offline di Management Console. Dalam situasi ini, endpoint perlu di-decommission manual dari console.

    Skenario yang Memerlukan Decommissioning:

    • Uninstall saat koneksi internet terputus
    • Agent tidak merespon command dari console
    • Endpoint sudah tidak ada tapi masih muncul di console
    • Status endpoint menunjukkan "offline" padahal sudah di-uninstall

    Langkah-langkah Decommissioning:

    1. Login ke Management Console 

      URL: https://your-tenant.sentinelone.net
Navigate: Sentinels → [Select Endpoint]

    2. Locate the Affected Endpoint 

      Filter by: Endpoint name, IP address, atau status
Status: Biasanya menampilkan "Offline" atau "Disconnected"

    3. Decommission Process 

      Right-click endpoint → Actions → Decommission
Atau: Select endpoint → Actions menu → Decommission
Confirm: Centang "I understand this action cannot be undone"
Click: "Decommission"

    4. Verification 

      Check: Endpoint hilang dari Sentinels list
Status: Tidak muncul lagi di console
License: License slot tersedia untuk endpoint baru

    Alternative via API (Advanced):

    # Decommission via SentinelOne API
curl -X POST "https://your-tenant.sentinelone.net/web/api/v2.1/agents/actions/decommission" \
  -H "Authorization: ApiToken your-api-token" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "ids": ["agent-id-here"]
    },
    "data": {}
  }'

    Kapan Melakukan Decommissioning:

    • ✅ After Physical Uninstall: Setelah memastikan agent benar-benar terhapus dari endpoint
    • ✅ Endpoint Disposal: Ketika endpoint/server sudah tidak digunakan lagi
    • ✅ Migration: Sebelum install ulang atau migrasi ke tenant lain
    • ❌ Active Endpoint: Jangan decommission endpoint yang masih aktif dan terkoneksi

    Praktik Terbaik

    • Instalasi Massal: Gunakan alat seperti SCCM atau Ansible
    • Uji Coba: Selalu uji di lab sebelum deploy ke produk
    • Dokumentasi: Catat semua endpoint yang diinstal
    • Pemantauan: Awasi resource setelah instalasi
    • Exclusion: Siapkan exclusion list sebelum instalasi
    • Backup Configuration: Simpan konfigurasi sebelum uninstall
    • Alternative Security: Pastikan ada proteksi alternatif sebelum uninstall

    Gunakan panduan ini untuk memastikan setiap langkah diimplementasikan dengan benar.
